The purpose of this External Module is to help guide you in renaming a username. Modifying a username might pose auditing problems, proceed with care!

It is your responsibility to triple check the SQL, Tables and Columns used by this External Module.

It is your responsibility to ensure that new usernames are compliant with REDCap.

Usernames can only contain letters, numbers, underscores, hyphens, and periods.

Create a backup of your database before continuing. Know how to restore from a backup.

Your database may have inconsistencies with the generated SQL script. An example is orphaned data in a child table that stops the SQL script from continuing. When updating multiple usernames it is possible that a database error stops the script from continuing and all usernames past a certain point are not updated. Information about the database collations and the affected tables are under the "Tables" and "DB Info" tab.

You preview the SQL before any changes are made. Review all code before proceeding! Double check all SQL code before running it.

The tabs above are here to provide you with information about your instance of REDCap and what this module will do.

1. Read Me: Overview of the External Module.

2. Tables: List of tables where the username will be updated.

3. Change User: GUI for changing a username.

4. Auth Method: Information about authentication methods used in your system.

5. DB Info: Information about your database collations and tables that will and will not be updated when a username changes.

6. Password info: Table based authentication information.
