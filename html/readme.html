This module's purpose is to help guide you in renaming a username. Modifying a username might pose auditing problems, proceed with care! Please proceed with caution.

It is your responsibility to ensure that new usernames are compliant with REDCap. Usernames can only contain letters, numbers, underscores, hyphens, and periods.

Your database may have inconsistencies with the generated SQL script. An example is orphaned data in a child table that stops the SQL script from continuing. Information about the database collations and the affected tables is under the "DB Info" tab.

The SQL is generated before any changes are made. Please review all code before proceeding! Double and triple check all SQL code before running it.

The following tabs are available:

1. Read Me: Overview of the External Module.

2. Change User: GUI for changing a username.

3. Auth Method: Information about authentication methods used in your system.

4. DB Info: Information about your database collations and tables that will be updated when a user name changes.

5. Password info: Table based authentication information.

6. Tables: List of tables where the username will be updated.
